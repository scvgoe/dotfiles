# 선언 순서 재배치 (Reorder Declarations)

파일 하나의 선언 순서를 추상화 수준에 따라 재배치합니다.
구현 자체는 변경하지 않고, 선언 위치만 조정합니다.

## 목표

- **Top-down 가독성**: 파일을 위에서 아래로 읽으며 점진적으로 deep-dive 가능
- **추상화 수준 정렬**: 높은 추상화 → 낮은 추상화 순서
- **구현 보존**: 로직 변경 없이 선언 위치만 이동

## 선언 순서 원칙

1. **타입/인터페이스 (exported)**
2. **상수 (exported)**
3. **메인 함수/클래스 (exported)** - 파일의 핵심 진입점
4. **헬퍼 함수 (exported)**
5. **내부 타입/인터페이스 (private)**
6. **내부 상수 (private)**
7. **내부 헬퍼 함수 (private)** - 호출되는 순서대로 배치

## 작업 절차

1. 대상 파일 경로를 사용자에게 확인
2. 파일 전체 내용 읽기
3. 각 선언을 카테고리별로 분류
4. 위 원칙에 따라 순서 재배치
5. 변경 전/후 구조를 사용자에게 보여주고 확인
6. 승인 후 파일 수정

## 주의사항

- 호이스팅이 필요한 경우 (함수가 선언 전에 참조되는 경우) 고려
- 순환 참조가 있는 경우 사용자에게 알림
- import 문은 파일 최상단 유지
- 기존 주석/JSDoc은 해당 선언과 함께 이동
